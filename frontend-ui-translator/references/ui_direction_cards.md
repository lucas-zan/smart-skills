# UI direction cards

Use these as anchors when the user says things like “make it nicer”, “more premium”, or “I don't know what looks good”. Pick 2-3 cards that fit the user's context. Adapt the wording; do not dump the entire file.

## 1. Precision SaaS
- Useful for: dashboards, admin panels, data tools, internal platforms
- Feels like: disciplined, efficient, credible, calm
- Layout pattern: strong page header, clear filter bar, dense-but-readable data region, optional side drawer
- Density: medium to medium-high
- Typography and color: neutral-heavy palette, one restrained accent, crisp hierarchy
- Component behavior: buttons and tables feel systematic, status chips are clear, empty/loading states are tidy
- Avoids: marketing-style gradients, playful decoration, random card nesting

## 2. Premium Product Marketing
- Useful for: landing pages, feature pages, pricing pages, product launches
- Feels like: polished, confident, premium, brand-aware
- Layout pattern: strong hero, proof/benefit rhythm, focused CTA cadence, sectional storytelling
- Density: low to medium
- Typography and color: bold type contrast, more whitespace, carefully limited accents
- Component behavior: fewer components, stronger section composition, screenshots/mockups matter
- Avoids: crowded hero sections, too many CTA variants, overexplaining every block

## 3. Calm Content-First
- Useful for: documentation, knowledge products, settings, editors, text-heavy tools
- Feels like: quiet, legible, thoughtful, trustworthy
- Layout pattern: content-first canvas, understated controls, clear reading rhythm
- Density: medium
- Typography and color: softer contrast, readable type scale, low-noise palette
- Component behavior: controls stay secondary to content, layout favors reading and editing flow
- Avoids: flashy effects, too many borders, aggressive emphasis everywhere

## 4. Premium Minimal
- Useful for: premium consumer products, polished onboarding, device/product showcases
- Feels like: spacious, refined, high-end, deliberate
- Layout pattern: large hero spacing, strong focal area, fewer competing elements
- Density: low
- Typography and color: generous whitespace, restrained palette, strong imagery or product focus
- Component behavior: motion and transitions should feel subtle and purposeful
- Avoids: cramped layouts, busy dashboards, decorative overkill

## 5. Commerce Utility
- Useful for: merchant tools, operations consoles, ecommerce admin, logistics tools
- Feels like: pragmatic, fast, operational, dependable
- Layout pattern: action-first toolbar, filter + list/table, clear bulk actions, detail panels
- Density: medium-high
- Typography and color: functional hierarchy, clear statuses, direct affordances
- Component behavior: efficiency matters more than ornament; support repetitive workflows
- Avoids: fancy effects that slow scanning, ambiguous action placement, hidden bulk actions

## 6. Creative Tool
- Useful for: builders, editors, collaboration products, design/dev tools
- Feels like: tool-like, flexible, sharp, power-user friendly
- Layout pattern: canvas plus inspectors, denser controls, multiple panes when justified
- Density: medium-high
- Typography and color: restrained interface chrome, content/canvas stays primary
- Component behavior: shortcuts, panel behavior, and state clarity matter more than decoration
- Avoids: soft decorative marketing treatments, oversized controls, vague states
